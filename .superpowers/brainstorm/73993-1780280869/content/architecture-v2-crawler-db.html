修订架构：采集管线 + 本地数据库 + 无账号系统

把“数字高人计划”拆成四个边界：采集、沉淀、蒸馏、评审。边界清楚，复杂度才不会发霉。

Revised System Map
01 Intake

Research Console

输入人物、关键词、种子 URL；配置采集深度、来源类型、黑白名单。

02 Pipeline

Crawler Worker

搜索发现、网页抓取、正文抽取、去重、来源可信度标记、失败重试。

03 Storage

Local DB

SQLite 保存人物、来源、资料片段、Skill、评审记录、任务状态。

04 AI

Proxy + LLM

后端读取 .env，调用 OpenAI 兼容模型，执行蒸馏与矩阵评审。

Material

原生家庭、教育、工作、投资组合、访谈、复盘、同行评价。

Skill

3-7 个心智模型、5-10 条启发式、表达 DNA、反模式、诚实边界。

Matrix

投决结论、对人判断、对事判断、风险清单、交叉批判链。

新增能力

  • 人物资料可以持续沉淀，不靠一次性粘贴。
  • 每条 Skill 结论能追溯到来源片段。
  • 评审历史可复盘，认知进化有轨迹。

首版边界

  • 采集遵守 robots 与速率限制。
  • 不登录第三方网站，不绕过付费墙。
  • 不做多用户权限，只有本机单人使用。
