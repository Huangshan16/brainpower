首版架构：本地后端代理 + AI 矩阵工作台

目标是把 API key、模型调用、蒸馏流程和前端体验分开，避免一个大组件吞掉整个产品。

System Map
Frontend

Matrix Console

项目输入、人物矩阵、Skill 编辑器、评审结果、交叉批判时间线。

Backend

AI Proxy

保存运行期配置、封装模型调用、构造 persona prompt、聚合评审任务。

Model

LLM Provider

OpenAI 兼容接口优先；后续可替换为 Anthropic、Gemini 或本地模型。

Skill Layer

人物资料 → 心智模型 → 决策启发式 → 表达 DNA → 反模式 → 诚实边界。

Evaluation Layer

投/不投、对事判断、对人判断、关键风险、尖锐质询、反驳链路。

取这个架构的理由

  • API key 不暴露在前端 bundle。
  • 模型供应商以后能替换。
  • 蒸馏和评审是两个清晰边界。

首版克制

  • 不做自动全网爬取，先支持粘贴资料和来源记录。
  • 不做账号系统，数据先存浏览器本地。
  • 不假装数字人物等于真人真实想法。
